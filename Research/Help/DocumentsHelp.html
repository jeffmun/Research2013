General Document Features

General Documents

The Autism Center Intranet Site provides that ability to store documents of several types which can be easily accessed from various locations on the web site. These documents are added to the web server in various ways, but most generally they are uploaded from the Documents section of the intranet site (see the section below on Uploading Documents for more information). Documents on the web site can fall into one of the following categories:

  • Consent Form Version
  • Data Form/Var Map
  • Admin Protocol
  • Behavior Summary
  • Misc Subject Document
  • Coding Manual
  • Reliability Info
  • Study Flow
  • Misc Measure Doc
  • Data Entry Instructions
  • Manual of Operations

Documents for DB Entities

There are several entities in the Autism Center Database that have related documents. The following list includes all of the types of entities for which we currently allow documents to exist on the web site:

  • Measures
  • Study Measures
  • Consent Forms (Versions)
  • Subjects
  • Households
  • Studies
  • Staff

Some examples of documents include Measure Variable Maps; Subject Behavior Summaries; Study MOPs; Data Entry Instructions; Consent Forms; etc. It is not required that document that is uploaded to the web server needs to be related to one of the entities listed above; however, some documents would make little sense outside the context of a relationship to an entity. For example, a Subject Behavior Summary is not useful unless we know the subject to whom it belongs.

Downloading Documents

In various places around the Autism Center Intranet Site, you will find links to these documents. For a complete list of documents on the web server, there is a report in the Reports section of the web site, under the Documents Reports category. While a page that allows staff members to search for a document does not currently exist on the intranet site at this time, there is a plan to build this in the future.

Uploading Documents

All members of the Autism Research Group and IT Staff Group have permission to view the Upload Document Page, and thus upload any document to the web server.

It is advised that you upload any document with extreme caution, as you are likely to overwrite an existing document.

Uploading New Documents

Even though you think you may be uploading a new document, it is very important to make sure that this document does not already exist on the web server before uploading. You can check this by referencing the Complete Document List Report in the reports section of the web site. Once you have confirmed that this document does not already exist, you can continue with the upload. If you find that the document indeed does exist check with the person who uploaded the document. If you need to edit this document and replace the existing one, see the next section below.

Editing Existing Documents

Sometimes it is necessary to edit a document that already exists on the web server. In order to do this, follow these steps:

  1. Make sure the changes you want to make are acceptble.
  2. Download the existing document to your PC (reference the Complete Document List Report)
  3. Open the file you just saved, make your edits to it.
  4. Click the "Browse" button on the Upload Document page and select the file you just edited.
  5. Fill out the rest of the Upload Document form.
  6. Click the "Upload" button.
